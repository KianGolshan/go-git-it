Go Git It

VS Code extension for vibe-coders — version control without the vocabulary.

2 · Extension Architecture

Core Modules

⚙️
extension.ts
Activation, command registration, state orchestration. Entry point VS Code calls on load.
🔧
gitRunner.ts
All Git subprocess calls. No VS Code imports — fully testable in isolation. Returns typed GitResult objects, never throws.
☁️
githubSetup.ts
GitHub repo creation via gh CLI. Falls back gracefully when CLI not installed.
📂
repoSwitcher.ts
Scans local dirs, queries recent workspaces, fetches GitHub repos. Merges into a unified QuickPick.
❓
errorExplainer.ts
Maps 4 error codes to warm plain-English copy. Renders a WebviewPanel — no raw Git output shown.
🛠️
uiHelpers.ts
Shared: withFriendlyProgress, showSuccess, slugify.

UI Layer

🌳
statusProvider.ts
TreeDataProvider for the Actions view. Renders status rows and all 9 action buttons. Updates on every git operation.
📊
treeWebview.ts
WebviewViewProvider for the Timeline view. Renders commit nodes as a visual storyboard with color-coded dots.
📌
Status Bar Item
Live badge: $(git-commit) N snapshots. Shown when a git repo is open. Clicks open the walkthrough.
😅
Error Explainer Panel
WebviewPanel (beside editor) with emoji hero, 3-section layout, accessible action buttons ≥44px.
📖
Walkthrough
4-step VS Code walkthrough registered in package.json. Covers snapshot, GitHub, experiments, rewind.
No backend required. All operations run locally via Git subprocess calls. GitHub connectivity uses the gh CLI — optional and gracefully degraded.

7 · Git Runner — Function Reference

GitResult contract Every function returns { ok, message, code?, rawError? }. Never throws. message is always plain English for the user. rawError stores original stderr and is only shown in the error explainer if the user clicks "Show technical details."
FunctionGit OperationsSuccess MessageError Code
takeSnapshot(cwd, summary?) git status --porcelain → git add -A → git commit "Snapshot saved ✅ — if something breaks, you can always come back to this point." NOTHING_TO_COMMIT
pushToRemote(cwd) git push "Your work is now backed up on GitHub ☁️" NO_UPSTREAM
pullLatest(cwd) git status → git fetch → git pull "Got the latest — your files are up to date ⬇️" or "You already have everything." DIRTY_PULL
createExperiment(cwd, name) git checkout -b experiment/[slug] "You're now experimenting safely 🧪 — nothing here affects your main work until you're ready." UNKNOWN
mergeExperiment(cwd, default) git checkout → git pull → git merge --no-ff "Experiment merged into your main line ✅" MERGE_CONFLICT
abandonExperiment(cwd, default) git checkout [default] → git branch -D "Experiment deleted — your main work is untouched." UNKNOWN
getState(cwd) git rev-parse git status git remote git log git log --remotes Returns GitState object (no user message) returns null if no .git
initProject(path, name) git init → write files → git add -A → git commit "Project created! Your first snapshot is saved." UNKNOWN
abortMerge(cwd) git merge --abort "Merge undone — your main work is back to normal." UNKNOWN

8 · Error Explainer — 4 Error States

⚠️ Raw Git output is never shown to the user. The error explainer translates every failure into warm, non-blaming plain English. A "Show technical details" disclosure is available for power users, but hidden by default.
CodeWhat We TriedWhat HappenedWhat To Do NextAction Button
DIRTY_PULL Get the latest version from GitHub You have edits that aren't saved as a snapshot yet — downloading could overwrite your work. Take a snapshot of what you have right now, then we'll get the latest. "📸 Snapshot then get latest" → takeSnapshot() then pullLatest()
MERGE_CONFLICT Merge your experiment into your main work Two sets of changes touched the same part of a file. Your work is completely safe. Undo the merge attempt and your work goes back to normal. "↩️ Undo the merge" → git merge --abort
NO_UPSTREAM Back up your work to GitHub This project isn't connected to a GitHub repository yet. Your snapshots are safe on this computer. Connecting to GitHub is a one-time setup. "🔗 Connect to GitHub" → GitHub setup flow
NOTHING_TO_COMMIT Save a snapshot There's nothing new to save — files haven't changed since the last snapshot. You're already up to date! Keep working and take a snapshot when you've made changes. (none — no action needed)

12 · Branch State Machine

no .git → git init → main (clean)
main (clean) → edit files → main (dirty) → take snapshot → main (committed) → send to GitHub → main (pushed)
main (clean) → start experiment → experiment/* (clean) → snapshot → experiment/* (committed)
experiment/* (committed) → finish experiment → main (merged)
experiment/* (committed) → finish experiment (conflict) → MERGE_CONFLICT → undo merge → main (clean)
experiment/* (any) → abandon experiment → main (clean) branch permanently deleted
main (dirty) → get latest → DIRTY_PULL → snapshot then pull → main (up to date)
main (committed) → send to GitHub (no remote) → NO_UPSTREAM → connect to GitHub → main (pushed)

13 · Installation & Setup

Installing Go Git It

1
Open VS Code or Cursor and go to the Extensions panel (⇧⌘X)
2
Search "Go Git It" and click Install
3
The Project Timeline icon appears in the Activity Bar immediately
4
Open any folder with a .git directory — the extension activates automatically
5
A first-run walkthrough launches with 4 friendly steps
6
No configuration required — works out of the box
GitHub CLI (optional): Install from cli.github.com to enable automatic GitHub repo creation. The extension works fully without it — GitHub connectivity is gracefully degraded.
