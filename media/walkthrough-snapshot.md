## Taking a snapshot

A snapshot is a save point — like saving a game before a tough level.

**When to take one:**
- Whenever something is working and you don't want to lose it
- Before trying something risky
- At the end of a work session

**What happens:**
Go Git It stages all your changed files and creates a commit with a timestamp. You can always come back to any snapshot using the timeline in the sidebar.

**Tip:** Give your snapshot a short description — it makes the timeline much easier to read later.
