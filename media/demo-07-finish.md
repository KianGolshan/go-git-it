## Step 7 — Finish or abandon the experiment

---

You've done the work. Now decide: keep it or scrap it.

---

### ✅ The experiment worked — merge it

Click **✅ Merge experiment** in the sidebar.

Go Git It will:
1. Switch back to your main line
2. Pull the latest from GitHub (in case anything changed)
3. Merge all your experiment snapshots into main
4. Return you to the main line with all the experiment's changes included

After a successful merge:
- The branch badge returns to 🌿 Main line
- The Merge / Abandon buttons disappear
- Your timeline shows all the experiment's snapshots merged in
- Send to GitHub to back everything up

---

### 🗑️ The experiment didn't work — abandon it

Click **🗑️ Abandon experiment** in the sidebar. Confirm the dialog.

Go Git It deletes the experiment branch entirely and returns you to your main line. Your main line is **exactly** as you left it when you started the experiment — not a single file changed.

---

### What if the merge causes a conflict?

Sometimes two changes touch the same part of a file. Go Git It will tell you clearly and show a **"Undo the merge"** button that safely aborts everything and returns your main line to normal.

You can then:
- Manually apply the experiment's changes one by one
- Or ask an AI assistant to help resolve the conflict

---

### After merging — take a snapshot and send to GitHub

```
✅ Merge experiment
   → 📸 Snapshot (optional description: "Merged hero redesign")
   → ☁️ Send to GitHub
```

This is the complete cycle.
