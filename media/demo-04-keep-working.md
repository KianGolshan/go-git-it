## Step 4 — Keep working and snapshotting

---

This is the everyday loop. Build something → snapshot → build something → snapshot.

Here's what a typical work session looks like:

---

### Example session

You're building a portfolio site. Here's how the afternoon goes:

| What you built | Snapshot description |
|---|---|
| Finished the About section | `About section done` |
| Added contact form | `Contact form — no validation yet` |
| Got the mobile layout working | `Mobile layout fixed` |
| Fixed the font sizes | `Font sizes adjusted` |
| End of session | *Send to GitHub* |

At the end of this, your timeline has 4 snapshots — all green after you send to GitHub.

---

### The status line tells you where you are

| Status | What it means |
|---|---|
| 🟡 You have unsaved changes | You've edited files since the last snapshot |
| 🔵 Saved here, not on GitHub yet | Snapshot taken, not sent to GitHub |
| ✅ Everything saved & backed up | Snapshot taken and on GitHub |

---

### The pulsing yellow dot

When the status shows 🟡, a pulsing yellow dot appears at the top of your timeline. This is a live indicator of unsaved work — it disappears the moment you take a snapshot.

---

### Pro tip: snapshot before breaks

Any time you're stepping away — lunch, end of day, going to a meeting — take a snapshot first. Future you will thank you.
